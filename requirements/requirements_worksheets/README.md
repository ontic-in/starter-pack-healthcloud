# Requirements Worksheets

Structured worksheets for requirement gathering sessions.

## Purpose

Document detailed requirements from structured requirement gathering meetings using worksheets that guide comprehensive analysis.

## Content Types

- Session notes from requirements meetings
- Feature breakdowns and analysis
- User story definitions
- Acceptance criteria templates
- User journey mappings
- Business process flows

## Naming Convention

- `Session_[NUMBER]_[TOPIC].md` - For session-based worksheets
- `[FEATURE]_Requirements.md` - For feature-specific requirements
- `[EPIC]_Analysis.md` - For epic-level analysis

## Example Files

- `Session_1_User_Analysis.md` - User personas and analysis
- `Session_2_Deep_Dive.md` - Detailed feature requirements
- `Omnichannel_Requirements_Session_Worksheet.md` - Specific feature deep dive
- `Reporting_Requirements_Session.md` - Reporting feature requirements

## Best Practices

1. Use structured worksheets for consistency
2. Include date and attendees in each worksheet
3. Link to related transcripts in `../client_call_transcripts/`
4. Extract user stories and create ClickUp tickets
5. Update worksheets as requirements evolve
