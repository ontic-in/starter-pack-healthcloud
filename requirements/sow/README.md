# Statements of Work (SOW)

Official contracts and statements of work.

## Purpose

Store all official contractual documents defining project scope, timeline, budget, and deliverables.

## Content Types

- Client-signed SOW documents
- Master Service Agreements (MSA)
- Scope definitions
- Timeline and milestone documents
- Budget and pricing documentation
- Change order documents

## Naming Convention

- `SOW_[CLIENT]_[PROJECT]_[VERSION]_[DATE].pdf`
- `MSA_[CLIENT]_[DATE].pdf`
- `ChangeOrder_[NUMBER]_[DATE].pdf`

Examples:
- `SOW_SIMU_Agentforce_v1.0_20250915.pdf`
- `MSA_SIMU_20250901.pdf`
- `ChangeOrder_001_20251015.pdf`

## Best Practices

1. **Version Control**: Include version numbers in filenames
2. **PDF Format**: Store signed contracts as PDF
3. **Markdown Summary**: Create `.md` summary for each SOW
4. **Change Tracking**: Document all scope changes with change orders
5. **Sign-off**: Ensure all parties have signed before storing
6. **Backup**: Keep both digital and physical copies
7. **Access Control**: Limit access to authorized personnel only

## SOW Summary Template

For each SOW PDF, create a corresponding `.md` file with key information:

```markdown
# SOW Summary: [Project Name]

**Client**: [Client Name]
**Project**: [Project Name]
**Version**: [Version Number]
**Date Signed**: YYYY-MM-DD
**Contract Value**: $[Amount]
**Duration**: [Start Date] to [End Date]

## Scope Summary

[High-level scope description]

## Key Deliverables

1. Deliverable 1 - Due: DATE
2. Deliverable 2 - Due: DATE
3. Deliverable 3 - Due: DATE

## Milestones

- Milestone 1: DATE - $AMOUNT
- Milestone 2: DATE - $AMOUNT
- Milestone 3: DATE - $AMOUNT

## Out of Scope

- Item 1
- Item 2

## Change Orders

- Change Order 001: DATE - [Description] - $AMOUNT
```
